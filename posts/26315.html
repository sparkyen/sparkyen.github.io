数论—模运算的世界

Posted on 2022-07-28 In 算法
介绍模运算的概念与性质，讲述乘法逆元及其求解方法，最后谈到如何使用中国剩余定理求解同余方程并证明其正确性

为什么需要求余数

在程序设计竞赛中，如果计算结果超出了64位整数的范围，则可能会要求输出结果对合适的数取模后的余数。这样可以消除在高精度计算方面，语言的差异所带来的不利因素。例如，Java中有 支持高精度计算的类Biglnteger，而C++和C则不得不靠自己实现。另一方面，高精度乘法不同实 现的复杂度也不一样，因此，对算法本身的评价变得更为困难。正因为此类理由，程序设计竞赛 中经常出现余数的计算 ## 模运算的基本概念与性质 ### 模的概念与性质 对于整数\(a,b\)，满足\(b>0\)，则存在唯一的整数\(q,r\)，满足\(a=bq+r\)，其中\(0\leq r < b\)。其中称\(q\)为商，\(r\)为余数。余数用\(a\ mod\ b\)或者\(a\%b\)表示。 1. \((a+b)\mod M=(a\mod M+b\mod M)\mod M\) 2. \((a-b)\mod M=(a\mod M-b\mod M)\mod M\) 3. \((a\times b)\mod M=(a\mod M\times b\mod M)\mod M\)

（温馨提示：先学习后面的乘法逆元再回来看下面的内容）

那么下面来看看，能不能进行“模除法”？例如\(5/2\equiv ?\pmod 7\)。

我们这样来看，在实数的世界里，如果 \(x=5/2\) ，那么就有 \(2\times x=5\) 。

类比地，在模7的运算的世界里，如果 \(5/2\equiv y\pmod 7\)，那么也应该有 \(2\times y\equiv 5\pmod 7\)，可以得到这个值为6。由此可以见是可以进行“模除法”的，那么该如何求模意义下的除法呢？

对于模意义下的除法我们可以利用乘法逆元将其转化为乘法。先将\(b\)在模\(M\)的意义下的逆元求出来，再与\(a\)相乘，再模\(M\)即可

  1. \((\frac{a}{b})\mod M = (a\mod M\times b^{-1}\mod M)\mod M\)

同余的概念与性质

若两数\(x,y\)除以\(M\)的余数相等，则称\(x,y\)模\(M\)同余，记做\(x\equiv y\pmod M\)

以下是几个最基础的性质： 1. 自反性：\(x\equiv x\pmod M\) 2. 对称性：若\(x\equiv y\pmod M\)，则\(y\equiv x\pmod M\) 3. 传递性：若\(x\equiv y\pmod M, y\equiv z\pmod M\)，则\(x\equiv z\pmod M\)

根据上述取模运算的计算方法，同余还有以下性质： 4. 同加性：若\(x\equiv y\pmod M\)，则\(x+z\equiv y+z\pmod M\) 5. 同乘性：若\(x\equiv y\pmod M\)，则\(x\times z\equiv y\times z\pmod M\) 6. 同幂性：若\(x\equiv y\pmod M\)，则\(x^n\equiv y^n\pmod M\) 7. \(x\times y\pmod M \Leftrightarrow M | (x-y)\) ## 乘法逆元 在实数意义下，对于任意\(a \neq 0\)，都有\(a \times \frac{1}{a}=1\)，\(\frac{1}{a}\)称作是\(a\)的倒数。所以在模意义下，也希望\(\frac{1}{a}\)满足这样的性质：\(a \times \frac{1}{a}\equiv 1(\mod\ p)\)。也就是说，尝试找到一个整数\(x\)，满足\(a \times x\equiv 1(\mod\ p)\)

若\(a \times x\equiv 1(\mod\ p)\)，则称\(x\)为\(a\)在模\(p\)意义下的乘法逆元，记为\(a^{-1}(\mod\ p)\)

接下来介绍两种方法来求出\(a\)在模\(p\)意义下的逆元

方法1：扩展欧几里得算法 从\(a \times x\equiv 1(\mod\ p)\)这个式子入手发现，它和\(\exists y,\ ax+py=1\)等价，可以使用扩展欧几里得算法解出这个关于\(a,p\)的不定方程。并且由裴蜀定理可以发现存在\(a\)在模\(p\)意义下的逆元当且仅当\(gcd(a, p) = 1\)，即\(a\)和\(p\)互质（\(a\perp p\)）。 

int inv(int a, int p){
	int x, y;
	exgcd(a, p, x, y);
	return x;
}

提示 逆元有时求出来可能是负数。例如若\(a=2,p=3\)，求\(a\)的逆元。 即求\(x\)使得\(2x\equiv 1\pmod 3\)，等同于\(2x+3y=1\)，其中一组解为\((-1,1)\)，此时-1就是\(a\)的逆元

方法2：费马小定理 费马小定理：若\(p\)是质数，且\(a\)不是\(p\)的倍数，则\(a^{p-1}\equiv 1(\mod\ p)\)

那么假如\(p\)是质数，就可以知道如果\(a\)不是\(p\)的倍数，就有\(a \times a^{p-2}=a^{p-1}\equiv 1(\mod\ p)\)。根据乘法逆元的定义，就知道\(a\)在模\(p\)意义下的逆元是\(a^{p-2}\)，可以用快速幂求出。 

int inv(int a, int p){
	return qpow(a, p-2, p);
}

线性求逆元

求出 \(1,2,...,n\) 中每个数关于 \(p\) 的逆元。

如果对于每个数进行单次求解，以上两种方法就显得慢了，很有可能超时，所以下面来讲一下如何线性（\(O(n)\)）求逆元。

递推： 假设现在已经知道了\(1 \sim i-1\)的逆元，现在想要求\(i\)的逆元

递推的初始情况：当\(i=1\)时，\(i^{-1} \equiv 1(\mod\ p)\)。接下来考虑\(2\leq i < p\) 的情况

我们令 \(k = \lfloor \frac{p}{i} \rfloor\)，\(j = p \bmod i\)，有 \(p = ki + j\)。再放到 \(\mod p\) 意义下就会得到：\(ki+j \equiv 0 \pmod p\)；

两边同时乘 \(i^{-1} \times j^{-1}\)（\(j<i\Rightarrow j^{-1}\)存在）： \(kj^{-1}+i^{-1} \equiv 0 \pmod p\) \(i^{-1} \equiv -kj^{-1} \pmod p\) 再带入 \(j = p \bmod i\)，有 \(p = ki + j\)，有： \(i^{-1} \equiv -\lfloor\frac{p}{i}\rfloor (p \bmod i)^{-1} \pmod p\)

我们注意到 \(p \bmod i < i\)，而在迭代中我们完全可以假设我们已经知道了所有的模 \(p\) 下的逆元 \(j^{-1}, j < i\)。

故我们就可以推出逆元，利用递推的形式实现：

\[ i^{-1} \equiv \begin{cases} 1, & \text{if } i = 1, \\ -\lfloor\frac{p}{i}\rfloor (p \bmod i)^{-1}, & \text{otherwises}. \end{cases} \pmod p \]

inv[1] = 1;
for(int i = 2; i <= n; i++)
	//防止出现负数
	inv[i] = p-inv[p%i]*(p/i)%p;

线性求任意 n 个数的逆元

上面的方法只能求 \(1\) 到 \(n\) 的逆元，如果需要求任意给定 \(n\) 个数（\(1 \le a_i < p\)）的逆元，就需要下面的方法：

首先计算 \(n\) 个数的前缀积，记为 \(s_i\)，然后使用快速幂或扩展欧几里得法计算 \(s_n\) 的逆元，记为 \(sv_n\)。

因为 \(sv_n\) 是 \(n\) 个数的积的逆元，所以当我们把它乘上 \(a_n\) 时，就会和 \(a_n\) 的逆元抵消，于是就得到了 \(a_1\) 到 \(a_{n-1}\) 的积逆元，记为 \(sv_{n-1}\)。

同理我们可以依次计算出所有的 \(sv_i\)，于是 \(a_i^{-1}\) 就可以用 \(s_{i-1} \times sv_i\) 求得。 

s[0] = 1;
for (int i = 1; i <= n; ++i) s[i] = s[i-1]*a[i]%p;
sv[n] = qpow(s[n], p-2);
for (int i = n; i >= 1; i--) sv[i-1] = sv[i]*a[i]%p;
for (int i = 1; i <= n; i++) inv[i] = sv[i]*s[i-1]%p;
## 同余方程与中国剩余定理 形如  \(ax\equiv c(\mod\ b)\)的方程被称为 线性同余方程

而在中国古代就有这类问题的踪影： > 有物不知其数，三三数之剩二，五五数之剩三，七七数之剩二。问物几何？

即求满足以下条件的整数：除以3余2，除以5余3，除以7余2。可以转写成下面这样一个方程组： \[ \begin{cases} x &\equiv 2 \pmod {3} \\ x &\equiv 3 \pmod {5} \\ x &\equiv 2 \pmod {7} \\ \end{cases} \]

该问题最早见于《孙子算经》中，并有该问题的具体解法。上面具体问题的解答口诀由明朝数学家程大位在《算法统宗》中给出：

三人同行七十希，五树梅花廿(niàn)一支，七子团圆正半月，除百零五便得知。

求解同余方程组—中国剩余定理

一般来说，中国剩余定理(Chinese Remainder Theorem, CRT)可以用来求解下面这样一类方程组： \[ \begin{cases} x &\equiv a_1 \pmod {m_1} \\ x &\equiv a_2 \pmod {m_2} \\ &\vdots \\ x &\equiv a_n \pmod {m_n} \\ \end{cases} \]

其中，\(m_1,m_2, \cdots, m_n\)两两互质，则对于任意的整数\(a_1,a_2, \cdots, a_n\)，上面这个方程组有解。中国剩余定理给出了 解的构造方式： 1. 计算所有模数的积\(M=m_1\times m_2\times \cdots\times m_n=\prod_{i=1}^nm_i\)。 1. 对于第i个方程计算： i. \(M_i =\frac{M}{m_i}\) ii. 计算 \(M_i\) 在模 \(m_i\) 意义下的逆元 \(M_i^{-1}\)（易知\(M_i\) 和\(m_i\)是互质的，所以逆元一定存在） iii. 计算\(b_i=M_iM_i^{-1}\)（不要对 \(m_i\) 取模） 3. 方程组的唯一解为：\(x=\sum_{i=1}^n a_ib_i \pmod M\)。

可以先来验证一下上面问题的解： 1. \(M=3\times 5\times 7=105\)； 2. 三人同行 七十 希：\(m_1=3, M_1=M/m_1=35, M_1^{-1}\equiv 2\pmod 3\)，故 \(b_1=35\times 2=70\)； 3. 五树梅花 廿一 支：\(m_2=5, M_2=M/m_2=21, M_2^{-1}\equiv 1\pmod 5\)，故 \(b_2=21\times 1=21\)； 4. 七子团圆正 半月：\(m_3=7, M_3=M/m_3=15, M_3^{-1}\equiv 1\pmod 7\)，故 \(b_3=15\times 1=15\)； 5. 所以方程组的唯一解为 \(x\equiv 2\times 70+3\times 21+2\times 15\equiv 233\equiv 23 \pmod {105}\)。（除 百零五 便得知）

下面我们来证明这个方法的正确性：

由于 1. 显然\(b_i=M_iM_i^{-1} \equiv 1\pmod {m_i}\) 2. 对于\(j\neq i\)，\(M_j\)为\(m_i\)的倍数，即\(M_j\equiv 0\pmod {m_i}\)。故\(b_j=M_jM_j^{-1} \equiv 0\pmod {m_i}\)

可以得到 \[ \begin{aligned} x&\equiv \sum_{j=1}^n a_jb_j &\pmod {m_i} \\ &\equiv a_ib_i+\sum_{j\neq i}^n a_jb_j &\pmod {m_i} \\ &\equiv a_i &\pmod {m_i} \end{aligned} \]

具体代码实现如下： 

ll crt(int n, ll* a, ll* r) {
  ll M = 1, x = 0;
  //M=r1*r2*...*rn
  for (int i = 1; i <= n; i++) M = M * r[i];
  for (int i = 1; i <= n; i++) {
	//mi=M/ri
    ll m = M/r[i], t, y;
    exgcd(m, r[i], t, y);  // m * t mod r[i] = 1
    x = (x + a[i]*m*t%M)%M;
  }
  return (x%M+M)%M;//保证x是正数
}

以洛谷-P1495 曹冲养猪为例说明其实际应用： 

#include <cstdio>
#define ll long long 
using namespace std;
const int maxn = 11;
int n;
ll r[maxn], a[maxn];
ll exgcd(ll a,ll b, ll &x, ll &y) {
	if (b==0) {
	    x = 1, y = 0;
	    return a;
	}
	ll d = exgcd(b, a % b, y, x);
	y -= (a/b)*x;
	return d;
}
ll crt(int n, ll* a, ll* r) {
  ll M = 1, x = 0;
  for (int i = 1; i <= n; i++) M = M * r[i];
  for (int i = 1; i <= n; i++) {
    ll m = M/r[i], t, y;
    exgcd(m, r[i], t, y);
    x = (x + a[i]*m*t%M)%M;
  }
  return (x%M+M)%M;
}
int main(){
	scanf("%d", &n);
	for(int i = 1; i <= n; i++)
		scanf("%d%d", &r[i], &a[i]);
	printf("%lld", crt(n, a, r));
}
